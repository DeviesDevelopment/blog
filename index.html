Introducing Workflow Timer

Workflow-timer is a Github action that measures the duration of the workflow and compares it to historical runs. The purpose of this action is to make the developer aware of when feedbacks loops get longer. Let’s say that you are running unit tests as part of your current workflow. If merging your changes (your PR) would increase the time it takes to run the unit tests by 50%, your changes probably have unwanted side effects.
